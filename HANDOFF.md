**HARDENING #1 (gentler rebalancing — reduce churn) DONE + verified, committed (NOT pushed — Venky reviews, then push). No migration.** The thinker used to re-buy toward exact target weights every run (high turnover, and it ignored existing holdings entirely). Now it computes the MINIMAL trade set:
- NEW pure module `lib/agent/rebalance.ts` → `planRebalance(input)`. Constants: `DRIFT_BAND = 0.05` (±5pp of total capital, absolute & explainable), `COOLDOWN_DAYS = 4` (covers ~3 trading days incl. a weekend).
  - (1) DRIFT BAND / hysteresis: a TARGET position is only traded when its weight deviates from target by >5pp; within band → left alone (kills daily micro-churn).
  - (2) PREFER CASH over selling: underweight targets are filled with available agent_cash first (best quant score first); a held name is only TRIMMED if it's >5pp OVER target, and a name no longer in the target set is fully EXITED.
  - (3) COOLDOWN: symbols the watchdog protective-sold within COOLDOWN_DAYS are not rebought. Inferred from `agent_decisions` where `action='sell'` (watchdog logs 'sell'; thinker trims/exits now log `action='trim'`, so 'sell' = watchdog only — NO new table/column needed). Skipped picks are LOGGED (`action='hold'`, "re-entry cooldown").
  - Guardrails enforced inside the planner: never spend below the cash buffer, never exceed max-position cap, never overspend agent_cash.
- `thinker.server.ts` rewired: reads current holdings + live prices, totalCapital = agent_cash + Σ(qty×price), filters cooldown out of targets (capturing dropped AI picks), calls `planRebalance`, executes (sells first then buys), seeds the trailing stop ONLY for brand-new positions (never lowers a ratcheted stop on an add). Rebalance decision rationale: "Portfolio within drift bands — no trades needed." when nothing material, else Claude commentary + "Adjusted N; held M within ±5pp band". New `runThinker(userId, { disableAi })` test seam (deterministic quant-only). Dashboard: new `trim`/`hold` decision-badge styling; run toast reflects trades vs no-op.
